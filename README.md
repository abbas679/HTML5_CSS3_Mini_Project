## Project Overview

This project represents a **modern personal portfolio website** that showcases my profile, skills, projects, and contact information. It emphasizes **responsive design**, **animations**, and **user-friendly navigation**.

Throughout this project, I structured my workflow into **13 incremental commits**, each demonstrating a specific skill or concept, from basic layout to advanced CSS techniques, including **flexbox, grid, pseudo-elements, accessibility improvements, and print styles**.

---

## Features

- Fully **responsive** design for mobile, tablet, and desktop.
- **Fixed navigation bar** with smooth hover effects.
- **Hero section** with animated headings and buttons.
- **About section** with profile image and styled text.
- **Skills section** with lists and tables, demonstrating flexbox and grid layouts.
- **Gallery section** with image overlay effects.
- **Media section** for video and audio content.
- **Contact form** with validation, accessibility, and styled inputs.
- **Footer** with social links, hover animations, and responsive layout.
- **Print-friendly styles** for document printing.
- **Accessibility improvements** including focus-visible, reduced motion, and proper semantic HTML.

## Learning Outcomes

Through this project, I gained practical experience in:

1. **HTML & CSS Fundamentals**

   - Structuring semantic HTML
   - Using headings, lists, forms, images, and links effectively

2. **Responsive Design**

   - Implementing media queries for different screen sizes
   - Flexbox and CSS Grid layouts for sections like skills, gallery, and media

3. **Advanced CSS Techniques**

   - Pseudo-classes (`:hover`, `:first-child`, `:nth-child`)
   - Pseudo-elements (`::before`, `::after`, `::first-letter`, `::first-line`, `::selection`)
   - CSS transitions and animations for buttons, images, and sections

4. **UI/UX Enhancements**

   - Creating hover and focus effects
   - Shadow, border, and outline usage
   - Consistent spacing, typography, and color schemes

5. **Accessibility & Best Practices**

   - `aria-labels` for buttons and links
   - Focus-visible outlines for keyboard navigation
   - Reduced motion for motion-sensitive users
   - Semantic HTML and meaningful `alt` attributes for images

6. **Version Control & Workflow**
   - Incremental commits to track project evolution
   - Organized commit messages demonstrating functionality growth
